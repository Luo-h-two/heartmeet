# 心遇 HeartMeet — 项目总结与简历描述

---

## 一、项目总结

### 1.1 项目概述

**心遇 HeartMeet** 是一个功能完备的相亲交友平台全栈项目，涵盖用户系统、推荐引擎、即时通讯、行为分析、AI情感助手等核心模块。前端采用纯原生 JavaScript 实现 SPA 架构，后端基于 Python FastAPI 异步框架，支持 PC 端（珍爱网式网格卡片）与移动端（探探式滑动卡片）双端自适应。

### 1.2 技术架构

| 层级 | 技术选型 |
|------|----------|
| **前端** | 原生 JavaScript (ES6+) SPA，手写路由/状态管理/组件化，CSS 变量 + 暗黑模式 |
| **后端** | Python 3 + FastAPI 异步框架 + SQLAlchemy ORM + Pydantic 校验 |
| **数据库** | SQLite（通过 aiosqlite 异步驱动） |
| **认证** | JWT Token（python-jose），SHA-256 + 随机盐值密码哈希 |
| **AI 集成** | DeepSeek API（OpenAI 兼容格式），异步 HTTP 调用 |
| **文件存储** | 本地文件系统，FastAPI StaticFiles 挂载 |

### 1.3 功能清单（共 17 个 API 端点 + 10 个前端页面）

#### 用户系统
- 手机号注册/登录（密码强度检测 + 表单校验）
- JWT Token 鉴权（24小时有效期）
- 个人资料 CRUD（昵称/年龄/性别/城市/职业/自我介绍）
- 择偶偏好设置（期望性别/年龄范围/城市）
- 头像上传（本地预览 + 服务器存储，支持 JPG/PNG/GIF/WebP，限 5MB）
- 相册管理（最多 9 张，上传/删除）

#### 推荐引擎
- **多维加权评分算法**：共同兴趣(+15)、同城(+20)、偏好匹配(+8~10)、活跃度(+5~10)、随机因子(+0~10)
- 支持分页、性别筛选、年龄范围、城市过滤、标签多选
- 每条推荐附带推荐理由（如"共同兴趣: 摄影、旅行，同城"）
- PC 端网格卡片布局 + 移动端探探风格大卡片

#### 社交互动
- 喜欢(♥) / 跳过(✕) / 打招呼(👋)
- 双向匹配检测（互相喜欢时弹出匹配成功动画）
- 聊天消息收发（私信）
- 消息列表（双向合并，去重排序）
- 已读/未读标记
- 2 秒消息轮询

#### 行为分析
- 行为总览：总浏览量、总喜欢数、总跳过数、喜欢率
- 偏好画像：性别偏好、年龄段偏好、标签 TOP5
- 数据可视化（进度条 + 柱状图 + 分布图）

#### AI 情商训练助手
- 两个模拟人格：小暖（女，活泼开朗） / 子辰（男，温和绅士）
- 详细系统提示词设计（性格特点、对话规则、回复风格）
- 预设 7 个场景聊天话题
- "正在输入"动画 + 消息气泡布局
- 最近 20 轮对话上下文，控制 Token 消耗

#### 交互体验
- 响应式双端适配（PC ≥1024px / 平板 768~1023px / 手机 <768px）
- 暗黑/浅色主题切换（CSS 变量体系）
- Toast 消息通知（成功/错误/提示）
- 大图预览、滑动卡片动画、匹配成功心跳特效

### 1.4 数据库设计（4 张表）

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `users` | 用户核心表 | 手机号、密码哈希、个人资料(JSON)、偏好(JSON)、照片(JSON) |
| `user_actions` | 交互行为 | like/skip/greet，双向匹配检测 |
| `behavior_logs` | 行为日志 | 浏览/停留/点击，大数据分析数据源 |
| `chat_messages` | 聊天记录 | 消息内容、已读标记、时间戳 |

### 1.5 种子数据

12 个模拟用户（覆盖不同性别/年龄/城市/职业/兴趣），预设交互行为，开箱即用。

## 二、简历项目描述

以下提供 **三种版本**，按适用场景选择：

---

### 版本 A：通用版（适合大多数技术岗位）

> **心遇 HeartMeet — 智能相亲交友平台** | 全栈开发
> 
> - 基于 **Python FastAPI** 异步框架 + **原生 JavaScript SPA** 构建全栈相亲交友平台，实现用户注册登录、个人资料管理、头像相册上传、即时通讯等核心功能
> - 设计并实现 **多维加权推荐算法**：综合兴趣标签匹配、同城加分、择偶偏好匹配、活跃度评分及随机扰动因子，为每个推荐附带可解释的推荐理由
> - 集成 **DeepSeek AI 大模型**（OpenAI 兼容 API），实现 AI 情商训练助手，通过精细化系统提示词工程模拟不同性别角色的人格化对话，辅助用户提升社交沟通能力
> - 实现 **用户行为数据采集与分析系统**：记录浏览、喜欢、跳过等交互行为，输出偏好画像报告（性别/年龄/标签偏好分布可视化）
> - 采用 JWT Token 鉴权 + SHA-256 密码哈希保障安全性，支持 PC/移动端双端响应式自适应布局及暗黑模式切换

**技术栈**：Python · FastAPI · SQLAlchemy · JavaScript · JWT · DeepSeek API · CSS3

---

### 版本 B：侧重后端/算法（适合后端开发或数据分析岗）

> **智能相亲推荐系统 — HeartMeet** | 后端开发
> 
> - 使用 **Python FastAPI + SQLAlchemy ORM + aiosqlite** 构建高性能异步后端，设计 17 个 RESTful API 端点，覆盖用户、推荐、交互、聊天、分析 5 大模块
> - 设计 **加权多因子推荐引擎**：基于标签交集的协同过滤思想，结合地理位置、偏好匹配、活跃度等因子，通过可配置权重实现个性化推荐，支持分页筛选
> - 集成 **大语言模型（DeepSeek）** 实现 AI 对话助手，通过角色设定式 Prompt Engineering 构建具有独立人设的虚拟对话人格
> - 设计 **行为日志系统**，采集用户多维交互数据（浏览停留时长、操作类型、偏好倾向），实现用户偏好画像分析与可视化展示
> - 使用 **Pydantic 数据校验** + JWT 认证 + 文件上传校验，保障接口安全与数据一致性

**技术栈**：Python · FastAPI · SQLAlchemy · SQLite · Pydantic · JWT · LLM 集成

---

### 版本 C：侧重大前端/全栈（适合前端或全栈开发岗）

> **HeartMeet — 响应式交友 SPA 全栈平台** | 全栈开发
> 
> - 使用 **纯原生 JavaScript (ES6+)** 从零实现 SPA 架构：手写路由系统、全局状态管理、组件化页面渲染，无任何前端框架依赖
> - 实现 **PC/移动双端自适应交互**：PC 端网格卡片布局（类珍爱网体验），移动端探探风格滑动卡片（Touch 事件 + 飞卡动画）
> - 基于 **CSS 变量 + 媒体查询** 实现完整的设计令牌体系和暗黑模式切换，覆盖 30+ 组件样式
> - 后端采用 **FastAPI + SQLite** 提供 RESTful API，实现即时通讯（消息轮询/已读标记/双向匹配弹窗）、图片上传相册管理、行为数据分析可视化
> - 前后端解耦部署：后端 FastAPI + uvicorn 服务端口 8000，前端静态文件服务端口 5173

**技术栈**：JavaScript · FastAPI · Python · CSS3 · HTML5 · SQLite · RESTful API

> **心遇 HeartMeet** | 全栈开发 | Python + FastAPI + 原生 JS

## 三、项目亮点（面试可用）

1. **推荐算法可解释**：每个推荐结果附带推荐理由，不仅告诉用户"谁合适"，还说明"为什么合适"
2. **AI 人格化对话**：通过精心设计的系统提示词，让 AI 展现出独立的性格特征和对话风格，模拟真实社交场景
3. **行为数据闭环**：用户行为 → 日志采集 → 画像分析 → 优化推荐，形成完整的数据驱动循环
4. **无框架前端**：纯 JS SPA 架构展示了扎实的 JavaScript 基础功底和架构设计能力
5. **全栈独立完成**：从数据库设计、后端 API、AI 集成、推荐算法到前端 UI，完整的产品开发能力
