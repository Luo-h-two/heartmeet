# 3. 路由设计

## 路由架构

心遇前端采用**纯前端路由**方案，不依赖框架路由库，通过 `navigateTo()` 函数实现页面切换。

## 路由表

| 路由名 | 路径标识 | 页面 | 是否需要登录 | 底部导航 |
|--------|----------|------|-------------|----------|
| login | `login` | 登录页 | 否 | 隐藏 |
| register | `register` | 注册页 | 否 | 隐藏 |
| home | `home` | 推荐页 | 是 | 推荐 ✓ |
| filter | `filter` | 发现页 | 是 | 发现 ✓ |
| chat | `chat` | 消息页 | 是 | 消息 ✓ |
| profile | `profile` | 个人中心 | 是 | 我的 ✓ |
| analysis | `analysis` | 行为分析 | 是 | 隐藏 |

## 路由守卫逻辑

```javascript
// 在每个需要登录的页面中检查
if (!Store.isLoggedIn()) {
  navigateTo('login');
  return;
}
```

- 未登录访问任何页面 → 自动跳转到登录页
- 登录后访问 login/register → 导航栏按钮切换
- 底部导航根据页面状态显示/隐藏

## 页面注册机制

```javascript
// 页面注册
registerPage('home', () => {
  // 渲染逻辑
});

// 页面跳转
navigateTo('home', optionalData);

// 当前页面状态
Store.currentPage  // 当前页面名称
```

## 导航组件

### 顶部导航栏
- 品牌 Logo（心遇 💕）
- 主题切换按钮（🌙/☀️）
- 登录/注册/退出按钮（根据登录状态动态切换）

### 底部导航栏（移动端）
- 4 个 Tab：推荐、发现、消息、我的
- 当前页面高亮（粉色）
- 消息 Tab 带未读红点
- 固定在页面底部，适配安全区域（safe-area-inset）
