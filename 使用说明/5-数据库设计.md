## 表设计说明

### 1. users 表
- **核心表**，存储所有用户信息
- `interests` 和 `personality_tags` 使用 JSON 类型存储数组，便于灵活扩展
- `preference` 使用 JSON 类型存储择偶偏好对象
- 支持地理位置（lat/lng），为后续 LBS 推荐预留

### 2. user_actions 表
- 记录用户之间的**交互行为**
- `action_type` 支持：like（喜欢）、skip（跳过）、favorite（收藏）、greet（打招呼）
- 双向喜欢检测：查询是否存在 `A→B` 的 like 且 `B→A` 的 like

### 3. behavior_logs 表
- **行为日志表**，用于大数据分析
- 记录用户所有操作：浏览卡片、点击详情、停留时长等
- `duration_ms` 记录停留时长（毫秒）
- `extra` 存储额外上下文信息

### 4. chat_messages 表
- 聊天消息记录
- 支持已读/未读状态
- 为完整聊天功能预留

## 数据库选择

项目使用 **SQLite** 作为数据库：
- 零配置，无需安装数据库服务
- 数据存储在 `heartmeet.db` 文件中
- 适合原型开发和演示

## 数据初始化

启动后端时自动执行：
1. 创建所有数据表（`Base.metadata.create_all`）
2. 检查是否已有数据，无数据则填充 12 个模拟用户
3. 为每个用户生成随机的交互行为数据
