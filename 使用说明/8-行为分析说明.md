# 8. 行为分析说明

## 分析目标

通过对用户行为数据的采集和分析，构建用户偏好画像，为个性化推荐提供数据支撑。

## 数据采集

### 采集的行为类型

| 行为 | 说明 | 数据字段 |
|------|------|----------|
| `view_card` | 浏览推荐卡片 | user_id, target_user_id, timestamp |
| `click_detail` | 点击查看用户详情 | user_id, target_user_id, timestamp |
| `like` | 喜欢某个用户 | user_id, target_user_id, timestamp |
| `skip` | 跳过某个用户 | user_id, target_user_id, timestamp |
| `stay` | 在卡片上停留 | user_id, target_user_id, duration_ms |
| `greet` | 打招呼 | user_id, target_user_id, timestamp |
| `search` | 搜索筛选 | user_id, filters, timestamp |

### 数据采集流程

```
用户操作 → 前端上报 → POST /api/behaviors → behavior_logs 表
         → 前端上报 → POST /api/actions   → user_actions 表
```

## 用户画像构建

### 画像维度

通过分析用户 **like** 过的用户，统计以下维度：

```
用户画像 = {
  偏好性别: 统计 liked 用户的性别分布,
  偏好年龄段: 统计 liked 用户的年龄区间,
  偏好城市: 统计 liked 用户的城市分布,
  偏好标签: 统计 liked 用户的兴趣/性格标签分布,
  行为特征: 喜欢率、浏览活跃度
}
```

### 自动标签生成规则

根据用户行为自动推断偏好标签：

```
if 用户经常 like 带"文艺"标签的用户:
    添加偏好标签 "文艺型"

if 用户经常 like 同城用户:
    添加偏好标签 "同城优先"

if 用户经常 like 比自己年龄小的用户:
    添加偏好标签 "偏好年轻"

if 用户喜欢率 < 20%:
    添加偏好标签 "挑剔型"
elif 用户喜欢率 > 60%:
    添加偏好标签 "广撒网型"
```

## 分析结果应用

分析结果被用于：

1. **推荐排序优化**：偏好标签匹配度高的用户排在前面
2. **推荐理由生成**：如"因为你常看文艺型用户"
3. **冷启动优化**：根据分析结果调整新用户的初始推荐
4. **用户洞察**：让用户了解自己的交友偏好

## 数据分析流程

```
1. 采集 → behavior_logs 表
2. 聚合 → 按用户维度统计
3. 分析 → 关联 liked 用户的属性
4. 输出 → 画像数据 + 可视化数据
5. 反馈 → 优化推荐策略
```

## 扩展方向

- **机器学习**：使用协同过滤、内容推荐等算法
- **实时分析**：流式处理行为数据
- **A/B 测试**：对比不同推荐策略的效果
- **用户分群**：聚类分析，划分用户类型
