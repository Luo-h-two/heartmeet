# 7. 推荐算法说明

## 算法架构

心遇推荐系统采用**多策略加权融合**方案，综合多种推荐维度计算匹配分数。

## 推荐策略详解

### 策略一：标签匹配（权重最高）

```
匹配分数 = 共同兴趣数 × 15 + 共同性格标签数 × 10
```

- 兴趣标签匹配（如两人都喜欢"旅行"、"摄影"）→ 每个 +15 分
- 性格标签匹配（如都是"开朗"、"文艺"）→ 每个 +10 分
- 择偶偏好标签匹配 → 每个 +8 分

### 策略二：地理位置加分

```
同城 → +20 分
```

- 相同城市的用户获得额外加分
- 为用户偏好城市列表中的用户加分

### 策略三：年龄偏好匹配

```
候选用户年龄在偏好范围内 → +10 分
```

- 根据用户设置的 `min_age` / `max_age` 判断

### 策略四：活跃度加分

```
有头像 → +5 分
有个人介绍（>10字） → +5 分
```

- 鼓励用户完善个人资料

### 策略五：随机因素

```
随机加分 0~10 分
```

- 避免推荐结果过于固化
- 保证一定的新鲜感和多样性

### 冷启动策略

新用户（无历史行为数据）：
- 默认推荐最活跃的用户（有头像、有个人介绍、有标签）
- 通过随机因素保证多样性

## 推荐算法伪代码

```
function getRecommendations(currentUser, filters):
    // 1. 过滤：排除自己和已操作过的用户
    candidates = allUsers
        .filter(u => u.id != currentUser.id)
        .filter(u => u not in actedUsers)
        .filter(u => matchFilters(u, filters))

    // 2. 计算匹配分数
    for each candidate in candidates:
        score = 0
        reasons = []

        // 标签匹配
        commonInterests = currentUser.interests ∩ candidate.interests
        commonTags = currentUser.personality_tags ∩ candidate.personality_tags
        score += len(commonInterests) × 15
        score += len(commonTags) × 10

        // 同城加分
        if candidate.city == currentUser.city:
            score += 20

        // 年龄偏好匹配
        if currentUser.preference.minAge <= candidate.age <= currentUser.preference.maxAge:
            score += 10

        // 活跃度加分
        if candidate.avatar: score += 5
        if len(candidate.bio) > 10: score += 5

        // 随机因素
        score += random(0, 10)

        candidates.scores.push({candidate, score, reasons})

    // 3. 按分数降序排列
    candidates.sortByScore(desc)

    // 4. 分页返回
    return paginate(candidates, page, pageSize)
```

## 推荐理由生成

每个推荐卡片附带推荐理由，让用户了解"为什么推荐这个人"：

| 条件 | 推荐理由示例 |
|------|-------------|
| 共同兴趣 | "共同兴趣: 摄影、旅行" |
| 性格相似 | "性格相似: 开朗、文艺" |
| 同城 | "同城" |
| 偏好匹配 | "符合你的择偶偏好" |
| 无特殊匹配 | "系统推荐" |

## 后续可优化方向

1. **协同过滤**：基于相似用户的喜好推荐
2. **行为加权**：停留时间长、反复查看的标签提高权重
3. **LBS 推荐**：基于地理距离排序
4. **时段推荐**：不同时段推荐不同类型的用户
5. **A/B 测试**：对比不同策略的匹配成功率
